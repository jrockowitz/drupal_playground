Understanding the Webform Module

The Webform module allows you to build any type of form to collect any type of data, which can be submitted to any application or system.

Learn more about Webform

Why Webform (and Drupal)

It provides all the features expected from an enterprise proprietary form builder combined with the flexibility and openness of Drupal.

The Use Case…

  • Build a form or copy a template.
  • Publish the form as a page, node, or block.
  • Collect form submissions.
  • Send confirmations and notifications.
  • Distribute submissions to external systems.
  • Review results online.
  • Download results as a CSV.

Building a Webform

  1. Create a feedback form.
  2. Test the form.
  3. View the submissions.
  4. Sending email
  5. Remote posting data
  6. Place the form as a block.
  7. Create a Webform node.

Webform Handlers

A Webform Handler is a plugin that processes a webform submission — essentially, it defines what happens after someone submits a form.

Webform Handler Use Cases

  • Email notification and confirmation
  • Push data to CRM.
  • Subscribe to mailing lists.
  • Flag a submission based on submitted values.
  • Conditionally change confirmation message.

Types of Webform Handlers

  • Email & Scheduled Email
    Sends a webform submission via an email
  • Action & Settings
    Trigger a conditional action or setting on a submission
  • Remote Post
    Posts webform submissions to a URL
  • Debug
    Debug webform submission
