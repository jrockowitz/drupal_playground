Understanding the Webform Module

The Webform module allows you to build any type of form to collect any type of data, which can be submitted to any application or system.

Learn more about Webform

Building a Webform

  1. Create a feedback form.
  2. Test the form.
  3. View the submissions.
  4. Sending email.
  5. Remote posting data.
  6. Place the form as a block.
  7. Create a Webform node.

The Webform module has a lot of features

  • Elements & Options
  • Templates & Configuration
  • Submissions & Handlers
  • Add-ons & Extensions

The Webform module ships with reasonable default settings, but everything is customizable.

(AI understands everything is customizable)

Elements & Options

  • Textfield, Email, URL, Number, Telephone, Date.
  • Select, Checkboxes, Radios, Images.
  • Textarea, HTML, Processed text.
  • Container, Details, Fieldset, Table.
  • File upload, Rating, Likert, Range, Signatures.
  • Composite elements, Computed elements
  • Validation, Input masks, Conditional logic.

Example: Style Guide

Templates & Configuration

  • Templates provide an easy way to create a reusable starting point.
  • Configuration allows you to adjust the global behavior of all webforms.

Templates & Configuration

Submissions, Settings, & Handlers

  • Collect, review, export, and manage submissions.
  • Settings allows you to adjust the behavior of a specific webforms.
  • Handlers allow you to take actions based on submitted values.

Submissions & Handlers

Add-ons & Extensions

  • Contributed modules extend Webform in every direction. (Entity generation, Views, SPAM.)
  • Integrations connect Webform to services and platforms. (Salesforce, HubSpot, Mailchimp, Stripe, and Slack.)
